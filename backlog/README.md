# Backlog Management

This folder is the source of truth for product and engineering backlog.

## Files
- `backlog/now-next-later.md`: prioritized roadmap buckets
- `backlog/product-backlog.md`: full feature backlog with acceptance criteria
- `backlog/tech-debt.md`: engineering quality and refactor items
- `backlog/release-checklist.md`: launch gates and go-live readiness

## Operating Rules
1. Every shipped PR must reference at least one backlog item.
2. Priorities are updated weekly.
3. "Now" items must have acceptance criteria and test notes.
4. Items move to done only after merged PR + verified behavior.
